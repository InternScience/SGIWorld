Research Agent Framework

MarkScientist

Self-evolving Research Agent with Built-in Scientific Taste

Proposer hypothesizes → Solver executes → Reviewer critiques → Iteration

Star on GitHub Quick Start

Key Features

Role Separation

Three-agent workflow with Proposer, Solver, and Reviewer, each with distinct responsibilities for systematic research.

JudgeBuddy System

15 research scenarios, 12 reviewer roles, and 5 scoring skills for comprehensive, scenario-aware evaluation.

Review-Driven Loops

Iterative improvement through automated review pressure until quality thresholds are met.

Taste Learning

Calibrates reviewer standards through user feedback to develop personalized scientific taste over time.

Scientific Workflow

1

Proposer

Generates research proposals with hypothesis, scope, and evaluation criteria. Reviewed before proceeding.

2

Solver

Executes the approved proposal using ResearchHarness. Produces artifacts like papers, code, or analyses.

3

Reviewer

Evaluates output with JudgeBuddy system. Provides scores, critiques, and verdicts (Accept/Revise/Repropose).

Workflow Complete

Iterates until the final score meets the threshold. Outputs include workflow trace and final artifacts.
